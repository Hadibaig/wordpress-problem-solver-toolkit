# Case 01 — Canonical URL Issue Affecting Rankings

**Category:** Technical SEO  
**Platform:** WordPress (RankMath / Custom Theme)  
**Difficulty:** Intermediate — Advanced  
**Time to Diagnose:** 40 minutes  
**Status:** ✅ Resolved

---

## ❗ The Symptom

Client noticed sudden drop in organic traffic.

Google Search Console showed:

- Duplicate pages indexed
- Wrong canonical URLs selected by Google
- Some pages missing from search results

---

## 🔍 Diagnosis Steps

**Step 1 — Checked page source**

Found multiple canonical tags:

```html
<link rel="canonical" href="https://example.com/page/" />
<link rel="canonical" href="https://example.com/page/?utm_source=..." />
```

---

**Step 2 — Checked SEO plugin (RankMath)**

RankMath was generating canonical URL automatically.

But theme was also adding custom canonical tag.

---

**Step 3 — Verified conflict source**

Search in theme files:

```php
add_action('wp_head', 'custom_canonical_tag');
```

So both:
- Theme
- SEO plugin

were outputting canonical tags.

---

**Step 4 — Confirmed Google behavior**

Google was ignoring signals due to duplicate canonical tags.

---

## 🎯 Root Cause

Duplicate canonical tags generated by:
- SEO plugin (RankMath)
- Custom theme function

This caused conflicting SEO signals.

---

## ✅ The Fix

Removed custom canonical function from theme:

```php
// Removed duplicate canonical tag function
// add_action('wp_head', 'custom_canonical_tag');
```

Kept only RankMath canonical output.

---

## 💡 Lesson

In SEO:

- Only ONE canonical tag should exist per page
- Never mix theme SEO logic with SEO plugins
